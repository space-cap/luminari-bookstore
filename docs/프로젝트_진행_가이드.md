# 서점 쇼핑몰 프로젝트 진행 가이드

이 문서는 Spring Boot를 사용하여 서점 쇼핑몰 프로젝트를 개발하는 과정을 단계별로 안내합니다.

## 1단계: 핵심 데이터 모델링 및 설정

가장 먼저 쇼핑몰의 핵심 데이터인 '책(Book)'을 관리하는 기능부터 구현합니다.

1.  **`Book` 엔티티 생성**:
    *   `com.ezlevup.luminari.domain` 패키지를 생성합니다.
    *   `Book` 클래스를 만들고 `@Entity` 어노테이션을 추가합니다.
    *   책의 속성(예: `id`, `title`, `author`, `price`)을 정의합니다.

2.  **`BookRepository` 생성**:
    *   `com.ezlevup.luminari.repository` 패키지를 생성합니다.
    *   `BookRepository` 인터페이스를 만들고 `JpaRepository<Book, Long>`를 상속받도록 합니다.

3.  **`BookController` 생성**:
    *   `com.ezlevup.luminari.controller` 패키지에 `BookController` 클래스를 생성합니다.
    *   `@RestController`와 `@RequestMapping("/api/books")` 어노테이션을 추가합니다.
    *   책을 등록, 조회, 수정, 삭제하는 CRUD API 엔드포인트를 구현합니다.
        *   `POST /api/books`: 새 책 등록
        *   `GET /api/books`: 모든 책 조회
        *   `GET /api/books/{id}`: 특정 책 조회
        *   `PUT /api/books/{id}`: 책 정보 수정
        *   `DELETE /api/books/{id}`: 책 삭제

4.  **테스트**:
    *   Postman이나 curl과 같은 도구를 사용하여 각 API가 올바르게 동작하는지 확인합니다.

## 2단계: 사용자 관리

책 관리 기능이 완성되면 사용자를 관리하는 기능을 구현합니다.

1.  **`Member` 엔티티 생성**:
    *   `com.ezlevup.luminari.domain` 패키지에 `Member` 클래스를 생성합니다. (`User`는 Spring Security에서 사용하는 클래스명과 겹칠 수 있어 `Member`를 추천합니다.)
    *   사용자 속성(예: `id`, `username`, `password`, `email`)을 정의합니다.

2.  **`MemberRepository` 생성**:
    *   `com.ezlevup.luminari.repository` 패키지에 `MemberRepository` 인터페이스를 생성합니다.

3.  **회원가입 및 로그인 기능 구현**:
    *   `MemberController`를 생성하여 회원가입 API를 구현합니다.
    *   **Spring Security**를 도입하여 비밀번호를 안전하게 암호화하고, 로그인 및 인증/인가 기능을 구현합니다.

## 3단계: 장바구니 기능

사용자가 책을 담을 수 있는 장바구니 기능을 구현합니다.

1.  **`Cart` 및 `CartItem` 엔티티 생성**:
    *   누구의 장바구니인지(`Member`), 어떤 책(`Book`)을 몇 개 담았는지(`quantity`) 등의 정보를 포함하는 엔티티를 설계합니다.

2.  **장바구니 기능 구현**:
    *   `CartController`를 생성하여 다음 API를 구현합니다.
        *   장바구니에 책 추가
        *   장바구니에 담긴 책 목록 조회
        *   장바구니에 담긴 책 수량 변경
        *   장바구니에서 책 삭제

## 4단계: 주문 관리

장바구니에 담긴 책을 실제로 주문하는 기능을 구현합니다.

1.  **`Order` 및 `OrderItem` 엔티티 생성**:
    *   주문 정보(주문자, 주문 날짜, 총 가격 등)와 주문된 상품 정보를 포함하는 엔티티를 설계합니다.

2.  **주문 기능 구현**:
    *   `OrderController`를 생성하여 다음 API를 구현합니다.
        *   장바구니 상품으로 주문 생성 (결제)
        *   주문 내역 조회

## 프론트엔드 (Frontend)

지금까지 구현한 API는 백엔드 기능입니다. 사용자가 실제로 웹사이트에서 기능을 사용하려면 프론트엔드 개발이 필요합니다.

*   **Thymeleaf**: Spring Boot와 통합이 쉬운 템플릿 엔진입니다. 서버 사이드 렌더링에 적합합니다.
*   **React / Vue.js**: 현대적인 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리/프레임워크입니다. API 서버와 연동하여 클라이언트 사이드 렌더링을 구현할 수 있습니다.
